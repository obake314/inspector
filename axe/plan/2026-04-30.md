# 作業ログ 2026-04-30

## 目的

BASIC SCAN の `color-contrast` が、重なり要素により「背景色を判定できませんでした / overlapped by another element」として `incomplete` になるケースを、スキャン用DOMの一時補正で再検査できるようにする。

## 対応

- `server.js`
  - BASIC / BATCH の通常 axe-core 実行後に、`color-contrast` の重なり由来 `incomplete` を抽出。
  - 該当セレクタだけに一時スタイルを当て、`color-contrast` ルールのみ再実行。
  - 再検査結果を `color-contrast-overlap-fallback` として `violations` / `passes` に追加。
  - 元の `incomplete` は削除せず、詳細パネルで根拠として残す。
  - テキストや操作要素など、装飾と断定できない重なり要素がある場合は補正せず未検証のまま残す。
  - 再検査後に一時スタイルと属性を削除。
- `public/index.html`
  - `color-contrast-overlap-fallback` の日本語表示名を追加。
- `spec/SPEC_WEB.md`
  - BASIC SCAN の補完再検査仕様を追記。

## 検証

- `npm test` 通過。

## 備考

この補完は本番ページを変更しない。Puppeteer 内の検査用ページに限定して適用し、BASIC完了時に補正を消去する。
