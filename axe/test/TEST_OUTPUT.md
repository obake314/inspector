## T-OUT-09: Excel 改善計画シート

- 手順

  2. `Excel` ボタンをクリック

  - `改善計画` シートのヘッダーが `優先度 / タイトル / 対象SC / スキャン元 / 概要・理由 / 改善手順` の6列である
  - `priorityActions` → `quickWins` → `manualChecks` の順で行が出力されている

## T-OUT-10: GoogleSheet 改善計画シート

- 手順
  1. MULTI SCAN を有効にして SCAN を実行し、改善計画が生成された状態にする
  2. GoogleSheet エクスポートボタンをクリック
  3. 生成されたスプレッドシートを開く
- 期待結果
  - スプレッドシートに `改善計画_{YYYY-MM-DD}_{HHMMSS}` という名前のシートが存在する
  - 改善計画シートは表紙シートの次・各ページシートの前に配置される
  - ヘッダー行が紺青（`#2D5799` 相当）背景・白字・太字でフォーマットされている
  - 列が `優先度 / タイトル / 対象SC / スキャン元 / 概要・理由 / 改善手順` の6列である
  - MULTI 改善計画がない場合は改善計画シートが生成されない

## T-OUT-11: 表紙スコアに改善計画行が混入しない

- 手順
  1. MULTI SCAN を有効にして SCAN を実行し、改善計画が生成された状態にする
  2. GoogleSheet エクスポートを実行する
  3. 生成されたスプレッドシートの表紙シートの「合格」列と、ページシートの「結果=合格」行数を比較する
  4. Excel エクスポートを実行し、各シートのデータを確認する
- 期待結果
  - 表紙の「合格」数 = ページシートの「結果=合格」行数（改善計画行を除く）
  - 表紙の「不合格」数 = 「結果=不合格」行数（改善計画行を除く）
  - 改善計画行（`検査種別=改善計画`）は結果列が空であり、表紙スコアの集計に含まれない
  - Excel の表紙集計でも同様に改善計画行が除外されている
