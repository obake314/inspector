# テスト要件 2026-04-23

## MULTI SCAN 関連
- [ ] `aiTarget: true` が付与された項目のみが、MULTI SCANの直接評価対象としてAPIペイロードに含まれることを確認する。
- [ ] BASIC, EXT, DEEP, PLAY の実行結果がデータクリーニングされた上で、MULTI SCANのバックエンドAPIに正しく送信されていることを確認する。
- [ ] MULTI SCANの結果に、他スキャンの検出項目に対する具体的な改善提案（improvementPlan）が含まれており、UI上に展開されることを確認する。
- [ ] 統合された詳細結果カードで、同じSC番号に対する各SCANの検出エビデンスが1つのカード内に重複なくまとまって表示されることを確認する。

## スコア計算 関連
- [ ] 各個別スキャン（BASIC, DEEP, MULTI, PLAY, EXT）の達成率が、`合格数 / (全項目数 - 未検証)` の計算式に基づいて算出・表示されることを確認する。
- [ ] TOTALスコアの分母が、選択した対象レベルの全項目数（Aなら31、AAなら56）で固定され、未検証や該当なしを差し引かないことを確認する。
- [ ] 個別スキャンによって評価されなかった項目が、TOTALスコアの集計において「未検証」として正しく計上されることを確認する。
- [ ] 詳細タブのバッジ数とTOTALスコアの内訳件数が一致していることを確認する。

## データパース・内部処理 関連 (2026-04-23 追記)
- [ ] AI応答のJSONパースにおいて、トップレベルのオブジェクトから `results` 配列と `improvementPlan` オブジェクトが欠落なく分離・抽出されることを確認する。
- [ ] サーバー側の内部デバッグ状態（`_lastAiDebug` 等）が、処理の進行に伴いフィールドを消失することなく適切にマージ・更新（例: `stage: 'ai_responded'`）されることを確認する。

## UI・エビデンス統合・BATCH 関連 (2026-04-23 追記)
- [ ] ツール読み込み時および「クリア」ボタン実行後、PLAYWRIGHT と EXT のオプションにデフォルトでチェックが入っていることを確認する。
- [ ] スコアテーブルの「全項目」列が、単一の数値ではなく `判定済み項目数 / 対象全項目数` のフォーマットで表示されることを確認する。
- [ ] 複数のスキャンから同一のエラー箇所やメッセージが検出された場合、詳細カード内でエビデンスが名寄せされ、1つのセクションにまとまって表示されることを確認する。
- [ ] MULTI スキャン実行後、不合格となった項目の詳細カード内に「AI改善案」がインラインで表示されていることを確認する。
- [ ] 一括検査（BATCH）を実行した際、SC 3.2.3 と SC 3.2.4 の詳細カードに `BATCH` バッジが付与されたナビゲーション一貫性の評価結果が統合表示されることを確認する。
- [ ] BATCH判定（合格/不合格）が、SC 3.2.3 / 3.2.4 において MULTI などの結果よりも優先して全体のステータスに適用されること（SC_AUTHORITYの動作）を確認する。
- [ ] PC+SP両方で一括検査を実行し、Excelエクスポートした際、SP側のシートにもナビゲーション一貫性の差分内容（テキスト形式）が欠落なく出力されることを確認する。
