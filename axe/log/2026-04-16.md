## 実施内容

### Fix 1: TOTALスコアの優先順位バグ修正

**問題**: DEEPで15件合格しているのにTOTALが5件合格しか表示されない。  
**原因**: `computeBasicScore` / `computeDeepScore` / `computeTotalScore` の rank 定義で `unverified` が `pass` より優先されていた（`{ fail:0, unverified:1, pass:2 }`）。  
**修正**: rank を `{ fail:0, pass:1, unverified:2, na:3 }` に変更（3関数すべて）。  
**結果**: SC単位マージ時に `pass` が `unverified` を正しく上書きするようになった。

---

### Fix 2: 対象レベルフィルタリングの適用漏れ修正

**問題**: レベル A 選択時に AA/AAA 項目が結果タブとレポート出力に混入していた。  
**原因**: `renderAllTabs` と `buildReportRows` でレベルフィルタが未実装だった。  
**修正**:
- `renderAllTabs`: BASIC violations/passes/incomplete に `getWcagLevel(tags)` ベースのフィルタを追加。DEEP results に `scSet.has(sc)` ベースのフィルタを追加。
- `buildReportRows`: 同様のフィルタを追加。DEEP の level 検出を hardcoded AAA set から `scLevelMap`（`WCAG_SC` 全体から生成）に変更。

### Fix 3: 詳細タブバッジ数とスコア表の不一致修正

**問題**: `.resultTabs` のタブ横数字がスコアテーブルの TOTAL 行と一致しない。  
**原因**: バッジは生の `ts.unverified` を使用、スコア表 TOTAL 行は `normalize()` で `unverified = fixedTotal - used` を算出していた。  
**修正**: `renderAllTabs` 内に `badgeMap` を追加し、スコア表と同じ正規化ロジックで unverified を算出。

### Fix 5: DEEP/MULTI スキャン完了後にスコア表が「未実行」のまま変わらない問題

### Fix 12: MULTI SCAN OpenAI 応答パース失敗によるスコア「未実行」バグ修正

**問題**: GPT-4o / o3 / GPT-5 を使用した MULTI SCAN が完了してもスコアテーブルの MULTI 行が「— 未実行 —」のまま変わらない。  
**原因**:
1. `callOpenAIAPI()` で `response_format: { type: 'json_object' }` を指定していたが、プロンプトがJSON配列（`[...]`）を要求しているため不整合。OpenAI はオブジェクト形式（`{"results": [...]}` 等）でラップして返し、サーバー側の配列パースが失敗 → 500 エラー → クライアントが `MULTI SCAN エラー` として処理し `snapshot.multi` が null → スコア「未実行」。
2. `o3` 等の推論モデルは `max_tokens` パラメーターをサポートせず `max_completion_tokens` が正しい。

**修正**:
- `server.js`: `callOpenAIAPI()` から `response_format: { type: 'json_object' }` を削除（プロンプトの指示に委ねる）
- `server.js`: `o3` / `o1` 系モデルは `max_completion_tokens: 8192` を使用、それ以外は `max_tokens: 4096`
- `server.js`: `/api/ai-evaluate` のJSONパース処理を改善—`Array.isArray` チェックを追加し、オブジェクトラップ形式 `{ results: [...] }` に対しても `Object.values().find(Array.isArray)` で配列を取り出せるように対応
